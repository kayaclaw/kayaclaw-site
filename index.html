$ what it does

Three moving parts. That is the whole agent.

  • 01

    Telegram interface

    You chat with the agent through your own Telegram bot. No web app to host, no front-end to maintain.

    interface · ~80 lines

  • 02

    SQLite memory

    Conversations live in one SQLite file on disk. Inspect it, back it up, or delete it with rm.

    storage · 1 file

  • 03

    Swap any provider

    Point it at DeepInfra, OpenRouter, Groq, DeepSeek, Qwen, or any OpenAI-compatible endpoint.

    router · config-driven

$ why it exists

A note from Anson.

OpenClaw is a fine project, but it is too large for me to audit comfortably in one sitting. NanoClaw nails the hardening philosophy I want, except it ties me to one provider and I want to point my agent at whichever LLM I trust this month.

kayaclaw is the version I wanted to read end to end before running on my own machine. One Dockerfile. One config file. If a claim in the README is not verifiable with a shell command, I removed the claim.

$ quick start

Three commands to a running agent.

Assumes Docker, a Telegram bot token, and an LLM API key.

bash · install + run 
$ git clone https://github.com/kayaclaw/kayaclaw
$ cd kayaclaw && cp config.example.toml config.toml
# point config.toml at your provider + bot token
$ docker compose up -d
$ docker logs -f kayaclaw
● ready. polling Telegram.
